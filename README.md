## Логика данных

- Учебная группа содержит студентов.
- Дисциплина является отдельной записью справочника и не принадлежит одной
  конкретной группе.
- Назначение дисциплины связывает дисциплину, группу, преподавателя, учебный
  год и семестр.
- Чтобы преподавать одну дисциплину нескольким группам, создаются несколько
  назначений дисциплины.
- Занятия создаются внутри назначения дисциплины.
- Оценка и посещаемость связывают студента с конкретным занятием.

## Импорт КТП

Преподаватель может импортировать календарно-тематический план в формате
`.xlsx` на странице назначенной дисциплины.

Поддерживаемые данные:

- порядковый номер;
- плановая дата;
- тема занятия;
- вид занятия;
- количество часов;
- литература.

После предварительного просмотра преподаватель подтверждает импорт и выбирает
темы КТП при создании занятий. Для зачёта используется система
`зачёт/незачёт`, для консультации оценивание отключено, для остальных видов
занятий применяется шкала `2–5`.

## Первый запуск

1. Установить Docker Desktop.
2. Создать локальный файл настроек:

   ```bash
   cp .env.example .env
   ```

3. Собрать и запустить приложение:

   ```bash
   docker compose up --build
   ```

4. Открыть в браузере `http://localhost:8000`.

## Демонстрационные данные

После первого запуска выполнить в отдельном окне терминала:

```bash
docker compose exec web python manage.py seed_demo
```

Будут созданы тестовая группа, дисциплина, занятие и пять студентов.

- Администратор: `admin` / `admin12345`
- Преподаватель: `teacher` / `teacher12345`

Демонстрационные пароли предназначены только для локального запуска.

## Обновление после изменения кода

Если изменились только файлы приложения:

```bash
docker compose restart web
```

Если изменились зависимости или `Dockerfile`:

```bash
docker compose up --build -d
```

## Остановка

```bash
docker compose down
```

Данные MySQL сохраняются в Docker volume и не удаляются при обычной
остановке контейнеров.

## Технические журналы

Приложение записывает в Docker технические события, HTTP-запросы,
предупреждения и ошибки. Просмотр последних записей:

```bash
docker compose logs --tail=100 web
```

Наблюдение за новыми событиями в реальном времени:

```bash
docker compose logs -f web
```

Для поиска ошибок:

```bash
docker compose logs web | grep -E "ERROR|CRITICAL"
```

Уровни журналирования задаются в `.env` параметрами `LOG_LEVEL`,
`DJANGO_LOG_LEVEL` и `GUNICORN_LOG_LEVEL`. Docker хранит не более пяти
файлов по 10 МБ для каждого контейнера.

Технические журналы не содержат оценок, ФИО и содержимого импортируемых
файлов. Изменения оценок и посещаемости фиксируются отдельно в защищённом
журнале изменений базы данных.

## Запуск готового образа на сервере

GitHub Actions собирает образ при каждом отправленном изменении в ветку
`main` и публикует его в GitHub Container Registry.

На сервере в `.env` нужно указать:

```dotenv
WEB_IMAGE=ghcr.io/github-user/college-journal:latest
```

Запуск и обновление:

```bash
docker compose -f docker-compose.prod.yml pull
docker compose -f docker-compose.prod.yml up -d
```
